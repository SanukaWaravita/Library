# 06 - Gantt Charts & Scheduling

## Overview

Gantt charts are visual project management tools that illustrate project schedules, showing activities, timelines, dependencies, and progress. They provide a clear overview of project timeline and resource allocation.

## Gantt Chart Fundamentals

### Definition and Purpose
- **Visual Timeline:** Graphical representation of project schedule
- **Activity Display:** Shows activities against time scale
- **Progress Tracking:** Visual progress indication
- **Resource Management:** Resource allocation visualization

### Key Components
- **Time Scale:** Horizontal axis showing time periods
- **Activities:** Vertical axis listing project activities
- **Bars:** Horizontal bars representing activity duration
- **Milestones:** Key project checkpoints and deliverables
- **Dependencies:** Relationships between activities

## Gantt Chart Creation

### 1. Activity Identification
**Activities:**
- Break down project into manageable tasks
- Define activity start and end dates
- Identify activity dependencies
- Estimate activity durations

**Deliverables:**
- Activity list with descriptions
- Duration estimates
- Dependency relationships
- Resource requirements

### 2. Schedule Development
**Activities:**
- Create project timeline
- Plot activities on time scale
- Add milestones and deadlines
- Include resource assignments

**Deliverables:**
- Preliminary Gantt chart
- Resource allocation plan
- Critical path identification
- Schedule baseline

### 3. Schedule Optimization
**Activities:**
- Identify resource conflicts
- Optimize resource utilization
- Adjust schedule for constraints
- Finalize activity sequencing

**Deliverables:**
- Optimized Gantt chart
- Resource leveling plan
- Schedule risk assessment
- Final schedule baseline

## Gantt Chart Elements

### Activity Bars
- **Length:** Represents activity duration
- **Position:** Shows start and end dates
- **Color Coding:** Different colors for activity types or status
- **Progress Indication:** Percentage complete visualization

### Dependencies
- **Finish-to-Start:** Activity B starts after Activity A finishes
- **Start-to-Start:** Activity B starts when Activity A starts
- **Finish-to-Finish:** Activity B finishes when Activity A finishes
- **Start-to-Finish:** Activity B finishes when Activity A starts

### Milestones
- **Key Events:** Major project achievements
- **Decision Points:** Critical decision moments
- **Deliverables:** Major output completion
- **Phase Gates:** Stage completion markers

### Resource Allocation
- **Resource Bars:** Show resource usage over time
- **Resource Histograms:** Resource utilization graphs
- **Overallocation Indicators:** Highlight resource conflicts
- **Resource Leveling:** Balance resource utilization

## Schedule Development Techniques

### Critical Path Method (CPM)
- **Network Analysis:** Activity dependency analysis
- **Critical Path:** Longest path through project
- **Float Calculation:** Activity scheduling flexibility
- **Schedule Compression:** Crashing and fast-tracking

### Program Evaluation and Review Technique (PERT)
- **Three-Point Estimation:** Optimistic, most likely, pessimistic
- **Expected Duration:** Weighted average calculation
- **Probability Analysis:** Schedule risk assessment
- **Monte Carlo Simulation:** Schedule uncertainty analysis

### Resource-Constrained Scheduling
- **Resource Availability:** Limited resource consideration
- **Resource Leveling:** Balance resource utilization
- **Resource Smoothing:** Minimize resource fluctuations
- **Schedule Adjustment:** Accommodate resource constraints
